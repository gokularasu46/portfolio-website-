## Challenges Faced

- Responsive design issues while adjusting layout for mobile screens.
- Card alignment issues in projects and certifications sections.
- Mobile layout problems due to limited screen width.
- Navigation design challenges while converting desktop menu into hamburger menu.
- Maintaining consistent spacing across sections.

## Solutions Implemented

- Used Flexbox for navigation, buttons, and footer alignment.
- Used CSS Grid for skills, projects, certifications, about, and contact layouts.
- Added media queries for 1024px, 768px, and 425px screen widths.
- Used responsive images with `max-width`, `aspect-ratio`, and `object-fit`.
- Added `overflow-x: hidden` and flexible containers to prevent horizontal scrolling.
- Used JavaScript to handle hamburger menu and smooth section navigation.

## Viva Questions and Answers

### 1. What is the purpose of your project?

The purpose of the project is to create a professional personal portfolio website that showcases my profile, skills, projects, education, certifications, and contact information.

### 2. Which technologies did you use?

I used HTML5 for structure, CSS3 for styling and responsiveness, JavaScript for interactivity, and GitHub for version control and hosting.

### 3. What is HTML5?

HTML5 is the latest standard of HyperText Markup Language. It is used to structure web pages using elements such as headings, paragraphs, sections, forms, links, and images.

### 4. What is CSS3?

CSS3 is used to style web pages. It controls colors, fonts, spacing, layout, animations, transitions, and responsive design.

### 5. What is JavaScript used for in your project?

JavaScript is used for the mobile hamburger menu, smooth scrolling, active navigation highlighting, section reveal animation, and contact form demo behavior.

### 6. What is responsive design?

Responsive design means creating a website that adjusts its layout and content based on different screen sizes such as desktop, tablet, and mobile.

### 7. What are media queries?

Media queries are CSS rules used to apply different styles for different screen widths or device conditions.

### 8. What is Flexbox?

Flexbox is a CSS layout method used to align items in rows or columns. It is useful for navigation bars, buttons, and flexible content alignment.

### 9. What is CSS Grid?

CSS Grid is a two-dimensional layout system used to create rows and columns. It was used for skills, projects, certifications, and contact layouts.

### 10. Why did you use a hamburger menu?

A hamburger menu saves space on small screens and provides a clean mobile navigation experience.

### 11. What is smooth scrolling?

Smooth scrolling allows the page to move gradually to a target section instead of jumping suddenly.

### 12. What are hover effects?

Hover effects are visual changes that happen when a user places the cursor over an element. They improve interactivity and user experience.

### 13. What animations did you use?

I used fade-in, slide-up, zoom-in, and smooth transition effects to improve the visual experience.

### 14. How did you prevent horizontal scrolling on mobile?

I used flexible widths, responsive grids, `max-width: 100%`, proper padding, and `overflow-x: hidden`.

### 15. What is the use of GitHub Pages?

GitHub Pages is used to host static websites directly from a GitHub repository and generate a public website link.

### 16. Why is semantic HTML important?

Semantic HTML improves readability, accessibility, SEO, and maintainability by using meaningful tags such as `nav`, `main`, `section`, and `footer`.

### 17. What is accessibility in web design?

Accessibility means designing websites that can be used by all people, including users who rely on keyboard navigation or assistive technologies.

### 18. What is the role of the contact section?

The contact section allows visitors, recruiters, or mentors to view contact details and send a message using the form.

### 19. What challenges did you face?

The main challenges were responsive alignment, mobile navigation, card layout consistency, and avoiding horizontal scrolling on small screens.

### 20. What future improvements can be added?

Future improvements include dark mode, backend integration, admin dashboard, AI chatbot, blog system, and resume download analytics.

## GitHub Upload Guide

### Step 1: Create Repository

1. Open GitHub.
2. Click **New Repository**.
3. Enter repository name: `Personal-Portfolio`.
4. Select Public.
5. Click **Create Repository**.

### Step 2: Upload Files

1. Open the repository.
2. Click **Add file**.
3. Click **Upload files**.
4. Upload `index.html`, `css/style.css`, `js/script.js`, images folder, documentation folder, and README file.
5. Click **Commit changes**.

### Step 3: Enable GitHub Pages

1. Go to repository **Settings**.
2. Select **Pages**.
3. Under Source, choose `Deploy from a branch`.
4. Select branch `main`.
5. Select folder `/root`.
6. Click **Save**.

### Step 4: Generate Website Link

After a few minutes, GitHub Pages will generate a link similar to:

```text
https://username.github.io/Personal-Portfolio/
```

This link can be used for final internship submission.
